# AI Stock Picker — AI 에이전트 한국 주식 Top 10 추천 시스템

> 한국투자증권(KIS) API로 시장 데이터를 수집하고, OpenAI Agents SDK 기반의 멀티 에이전트가
> 매일 두 가지 전략으로 코스피/코스닥 Top 10 종목을 선정하는 시스템입니다.
> 결과는 React 웹사이트로 배포됩니다.

### 🔗 라이브 사이트 : **https://hsh2578.github.io/ai-stock-picker/**

> ⚠️ **투자 권유가 아닙니다.** 이 프로젝트는 AI 에이전트·퀀트 파이프라인 학습을 목적으로 만든
> 개인 프로젝트입니다. 모든 종목 선정 결과는 참고용이며, 투자 판단과 책임은 본인에게 있습니다.

---

## 1. 어떤 프로젝트인가

매일 한국 주식 시장(코스피 + 코스닥) 약 1,000개 종목을 대상으로,
서로 다른 철학을 가진 **두 개의 독립된 AI 전략**이 각각 Top 10을 뽑아 나란히 비교합니다.

| 전략 | 이름 | 철학 | 핵심 정량 필터 |
|------|------|------|----------------|
| **전략 A** | 추세 추종 (노션 원본) | 이동평균선이 정배열로 전환되며 추세가 시작되는 종목을 잡는다 | MA 골든크로스 + 상대강도(RS) |
| **전략 B** | 테마/모멘텀 | 오늘 시장의 주도 테마에서 거래량이 터지며 초입에 있는 종목을 잡는다 | 거래량 급증 + 수급(외국인/기관) |

각 전략은 **정량 필터(코드)로 후보를 거른 뒤, AI 에이전트가 정성 분석으로 순위를 매기는**
2단계 구조입니다. 정량 필터가 "거를 자격이 있는 종목"을 정하고, AI는 그 안에서만 판단합니다.
(AI가 임의로 검증되지 않은 종목을 끌어오는 것을 막기 위한 설계)

└────────┬─────────────────────┘

┌─────────────────┐   두 전략 오케스트레이션, 결과 병합

## 3. 투자 유니버스 생성 (`kis_client.py`)

1. 한투에서 제공하는 **코스피·코스닥 종목 마스터 파일(.mst.zip)** 을 내려받아 파싱
2. 다음을 제외하여 **"순수 주식"** 만 남김:
   - 6자리 숫자 코드가 아닌 종목
   - ETP/ETN/ETF (플래그 + 브랜드 접두사: TIGER, KODEX, KINDEX, KBSTAR, SOL, ACE … + 키워드: 레버리지/인버스)
   - SPAC(스팩), 거래정지, 정리매매 종목
3. **시가총액 ≥ 2,000억 원** 필터 적용 → 약 1,000개 종목
4. 시총 내림차순 정렬하여 `(codes, symbol_map, universe_df)` 반환

> KIS API 세부: 토큰 23시간 캐싱, `RateLimiter(18)` 으로 초당 18건 제한,
> 일봉은 `FID_ORG_ADJ_PRC="1"` **수정주가**(액면분할·무상증자 반영)로 조회.
> 수정주가를 안 쓰면 가짜 골든크로스가 생기므로 반드시 "1".

## 4. 데이터 수집 & 캐싱 (`data_collector.py`)

`cache/ohlcv_meta.pkl` 의 날짜를 보고 3단계로 동작:

| 상황 | 동작 | 소요 |
|------|------|------|
| 오늘자 캐시 존재 | 디스크에서 즉시 로드 | ~0초 |
| 과거 캐시 존재 | 마지막 캐시일 다음날 ~ 오늘만 **증분 다운로드**, 신규 편입 종목은 250일 전체 | 2~3분 |
| 캐시 없음 | 유니버스 전체 250일치 **전체 다운로드** | 20~30분 |

- 병렬: `ThreadPoolExecutor(max_workers=8)` (Rate Limit 18/s 에 맞춰 튜닝)
- 증분 시 기존+신규 병합 후 중복 제거, 250일 윈도우 밖 데이터는 잘라냄
- 종가 0원·거래량 0인 비정상 행은 품질 검증에서 제거

## 5. 정량 스크리닝 도구 (`tools.py`)

AI 에이전트가 함수처럼 호출하는 `@function_tool` 들입니다.

### 5-1. MA 골든크로스 — `get_ma_cross_candidates` (전략 A)
- 단기 이동평균선이 장기 이동평균선을 **오늘 처음** 상향 돌파(`prev: short ≤ long`, `today: short > long`)
- 거래대금·거래량 순으로 정렬해 상위 N개 반환

### 5-2. 상대강도 RS — `get_relative_strength_candidates` (전략 A 공통)
- 최근 `lookback_days` 동안의 수익률 = `Close[-1] / Close[-lookback-1] - 1`
- RS 높음 = 시장 대비 강하게 상승. 점수 내림차순 상위 N개 반환

### 5-3. 거래량 급증 + 수급 — `get_volume_surge_candidates` (전략 B 핵심)
**테마 초입 종목**을 잡기 위한 다중 조건 필터:

| 조건 | 기본값 | 의도 |
|------|--------|------|
| 당일 상승률 | ≥ 5% | 확실히 올라간 종목만 (애매한 움직임 제외) |
| 거래량 급증 | ≥ 20일 평균의 2배 | 자금이 새로 유입됨 |
| 당일 양봉 | 종가 > 시가 | 장중 매도 폭탄(윗꼬리) 제외 |
| 거래대금 | ≥ 10억 원 | 유동성 노이즈 제거 |
| 최근 5일 수익률 | < 40~50% | 이미 꼭대기까지 간 종목 제외 |

→ 통과한 상위 후보에 **외국인/기관/개인 5일 합계 + 당일 순매수량**(KIS 투자자 동향 API)을 첨부.
AI가 "5일 추세 vs 오늘 신규 유입"을 직접 비교해 수급 전환을 판단할 수 있게 함.
이 전략에서 RS 필터는 일부러 노출하지 않음(AI가 검증 안 된 종목을 끌어오는 것 방지).

## 6. AI 에이전트 구조

두 전략 모두 **3-에이전트 체인**입니다. (OpenAI Agents SDK, 모델 `gpt-5-mini`, reasoning effort `medium`)

```
메인 픽커 에이전트 (Stock Picker)
├─ ① 매크로 에이전트 (as_tool)        → 오늘 장세 브리핑 + 필터 파라미터 제안
├─ ② 정량 도구 (function_tool)        → 후보군 계산 (MA / RS / 거래량급증)
├─ ③ 종목 조사 에이전트 (as_tool)     → 개별 종목 정성 리서치 (WebSearch)
└─ WebSearchTool                      → 메인이 직접 추가 조사
        ↓
   최종 Top 10 JSON 반환
```

### 전략 A — 추세 추종 (`strategy_original.py`)
노션 원본 책 설계를 충실히 구현.
- **매크로 에이전트**: 오늘 MA 돌파가 유리한지 / RS가 유리한지 판단하고 `short_window`,
  `long_window`, `lookback_days` 파라미터를 제안
- **종목 조사 에이전트**: 범용 리서치 — 사업 내용 / 최근 뉴스 / 긍정 요인 / 리스크
- **메인 픽커**: 매크로 브리핑 → 정량 필터로 후보 압축 → 애매한 종목만 추가 조사 → Top 10

### 전략 B — 테마/모멘텀 (`strategy_theme.py`)
- **매크로 에이전트**: 테마 장세인지 조정 국면인지 판단, 거래량 필터 파라미터
  (`surge_ratio`, `avg_days`, `min_change_pct`, `max_return_pct`) 제안
- **종목 조사 에이전트**: 테마/뉴스 모멘텀 집중 분석
  - 오늘 왜 올랐나(수주/실적/정책/뉴스/루머)
  - 재료가 이미 주가에 반영됐나, 후속 재료가 있나, 일회성인가 구조적인가
  - **대장주 판별** — WebSearch로 해당 테마 1등주를 실제 검색해 `is_top_leader` /
    `theme_leader_name` 판정
- **메인 픽커 순위 결정 기준 (중요도 순)**:
  1. **추가 상승 여력** (덜 반영 + 후속 재료 + 구조적이면 최상위)
  2. **수급** (외국인/기관 매수 > 개인만 매수)
  3. **촉매 강도** (정책/실적 > 뉴스 > 루머)
  4. **대장주 여부** (같은 테마면 1등주 우선)
  5. 리스크 대비 기대수익
  6. 진입 타이밍 (초입 > 이미 급등)
  7. 시총 대비 변동성/유동성
- 출력에 `market_summary`(오늘 시장 요약 2~3문장)를 함께 반환

## 7. 결과 병합 (`main.py`)

- 실행 인자로 전략 선택: `theme` / `original` / `both`(기본)
- **전략별 실행일 보존**: 기존 `cache/top10_result.json` 을 먼저 읽고,
  이번에 실행한 전략의 키만 덮어씀. `meta.strategy_run_dates` 에 전략별 마지막
  업데이트 시각 기록 → 웹은 전역 날짜가 아닌 **전략별 날짜**를 표시
- `both` 실행 시 두 전략에 **공통으로 포함된 종목**(강한 신호)을 별도 표시
- 캐시에서 실시간 종가·등락률·시가총액을 각 종목에 첨부
- `market_summary`는 테마 전략 실행 시에만 갱신(원본만 돌리면 기존 값 유지)

## 8. 웹 프론트엔드 (`web/`)

- **Vite + React 19 + TypeScript + Tailwind v4**, 라우팅은 `HashRouter`
  (GitHub Pages에서 서버 라우팅 없이 동작)
- `vite.config.ts` 의 `base: '/ai-stock-picker/'` 로 Pages 경로 대응
- 결과는 `import.meta.env.BASE_URL + 'data/top10_result.json'` 에서 fetch
- 페이지: 홈 / 추세 추종 전략 / 테마 전략 / 두 전략 비교
- `path` 가 배열·문자열 양쪽으로 올 수 있어 `getPathSteps`로 방어적 처리,
  `confidence` 도 `"78%"`/`78` 양쪽 처리
- 각 전략 페이지는 `meta.strategy_run_dates[...]` 우선, 없으면 `meta.run_date` 폴백

## 9. 실행 방법

### 사전 준비
프로젝트 루트에 `.env` 생성 (gitignore 됨):
```env
OPENAI_API_KEY=sk-...
KIS_APP_KEY=...
KIS_APP_SECRET=...
```

### 백엔드 (분석 파이프라인)
```bash
pip install -r requirements.txt

python main.py theme      # 테마/모멘텀만
python main.py original   # 추세 추종만
python main.py both       # 둘 다 (기본)
```
실행 비용 (gpt-5-mini 기준): theme ≈ 150~300원, original ≈ 100~300원, both ≈ 300~600원.

### 웹 (React)
```bash
cd web
npm install
npm run dev               # 로컬 개발 서버
npm run build             # web/dist/ 빌드
npx gh-pages -d dist      # GitHub Pages(gh-pages 브랜치) 배포
```
`main.py` 실행 후 결과를 웹으로 옮기기:
```bash
cp cache/top10_result.json web/public/data/
```

## 11. 알려진 한계

- **장중 실행 시** 오늘 일봉의 종가는 "현재가 스냅샷"이라 확정값이 아님.
  양봉/등락률 필터가 장중 매도세에 따라 뒤집힐 수 있음 → **확정 데이터는 15:30 KST 이후 실행**
- `Runner.run(max_turns=30)` — 테마 전략에서 조사할 종목이 너무 많으면 도달 가능.
  그 경우 후보 수를 줄여야 함
- 시가총액 2천억 미만 소형주는 유니버스에서 제외 (테마 반응은 크지만 본 시스템 범위 밖)
